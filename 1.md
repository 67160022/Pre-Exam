# ข้อที่ 1. สถาปัตยกรรมเว็บและ HTTP

**อธิบายกระบวนการสื่อสาร HTTP/HTTPS เมื่อผู้ใช้ป้อน URL ในเบราว์เซอร์ จากการส่งคำขอ (Request) ไปถึงการรับข้อมูล (Response) กลับมา รวมถึง บทบาทของ Status Code (เช่น
200, 404, 500) และเหตุใดการใช้ HTTPS จึงสำคัญสำหรับความปลอดภัย**

**ตอบ**
เมื่อผู้ใช้พิมพ์ URL ลงใน browser กระบวนการสื่อสารผ่าน HTTP/HTTPS จะเกิดขึ้นเป็นลำดับดังนี้:

1.DNS จะทำการแปลงชื่อเว็บเป็น IP โดย browser ก็คือ browser ส่งคำขอ ไป DNS แล้วแปลงชื่อ Domain ให้เป็น IP Address ของ server

2.สร้างการเชื่อมต่อโดย :
HTTP -> ใช้ TCP ปกติ
HTTPs -> มีการทำ TLS Handshake เพื่อเข้ารหัสก่อนสื่ิอสาร (TLS คือโปรโตคอลที่ใช้เข้ารหัสการสื่อสารบนอินเทอร์เน็ต)

3.ส่ง HTTP Request คือ browser ส่งคำขอไปยัง server เช่น `GET` เพื่อขอข้อมูลหน้าเว็บ พร้อมข้อมูลเสริม (headers,cookies)

4.server ประมวลผล ห
serveรับคำขอ -> ตรวจสอบ -> ดึงข้อมูล (เช่นจาก database) -> เตรียมผลลัพธ์

5.ส่ง HTTP Response เซิฟเวอร์ตอบกลับ โดยมี :

- status Code
- Headers
- ข้อมูล (HTML,JSON ...)

6. browser แสดงผล จะนำข้อมูลมา render เป็นหน้าเว็บให้ผู้ใช้เห็น
